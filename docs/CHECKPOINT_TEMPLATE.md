# CHECKPOINT_TEMPLATE — Werewolf-agent

每个 checkpoint 必须填写以下报告。目的是让非代码审查者也能判断项目是否真的前进。

---

```md
## Checkpoint Report — [日期] [checkpoint 编号]

### 1. 本轮用户可见变化

用户现在能看到什么？和上一个 checkpoint 相比有什么变化？

（必须描述可视化产出，不能只写"完成了 X 模块"。）

### 2. 页面路径、截图或录屏

必须至少提供一种可视化验收材料。没有时说明为什么本轮没有用户可见验收。

### 3. 固定输入输出

本轮的固定输入是什么？固定输出是什么？

（列出具体文件路径或数据样例。不要写"同上轮"——每轮独立可读。）

### 4. 新增或修改的指标

本轮新增了哪些评测指标？修改了哪些已有指标？

（引用 `@docs/EVALUATION_RUBRIC.md` 中对应的规则编号或指标名。）

### 5. deterministic 指标是否可复现

两次运行同一输入，所有 deterministic 指标结果一致？

（如不一致，说明哪个指标、什么原因。）

### 6. AI 标注记录（如有）

| 标注任务 | model | token_count | confidence | 一致性（3 次） |
|----------|-------|-------------|------------|---------------|
| — | — | — | — | — |

（Phase 1 如果本轮无 AI 标注，写"无 AI 标注，本轮全部 deterministic / gold / mock"。）

### 7. 数据标注清晰性

本轮产出中以下标注是否正确：

- [ ] `[结构化事件]` — 人工整理的事件
- [ ] `[deterministic]` — 确定性规则计算
- [ ] `[人工 gold sample]` — 人工编写的理想输出
- [ ] `[mock]` — 占位假数据
- [ ] `[AI 生成]` — 真实 AI 产出（Phase 1 预计全为"无"）

混淆了任何标注？（是/否，如是说明哪里）

### 8. 本轮产品价值是否增加

一句话回答：用户为什么更能感知评测系统的价值？

（不能只写"增加了指标"——要写"用户现在能看到 X"。）

### 9. 本轮不代表什么

明确不能宣称的能力。

（例如："本轮完成了确定性评分，但不代表 AI Agent 能力已被评测——Decision Log 和 Consensus Log 仍是人工 gold sample，decision_quality_score 恒为 0。"）

### 10. 下一步最大风险

如果继续，最可能偏掉的地方是什么？

（例如："下一步可能过度投入到可视化样式，应优先验证评分器规则无歧义。"）

### 11. 本轮任务对照

| 任务 ID | 状态 | 备注 |
|---------|------|------|
| S0 | pending/in_progress/completed | — |
| ... | | |

### 12. Stop / Review Gate 检查

- [ ] 是否连续两个任务没有用户可见变化？（是 → 停止，做 UX demo）
- [ ] 是否需要修改 EVALUATION_RUBRIC.md？
- [ ] 是否应砍范围或执行失败决策？
- [ ] 是否出现了"已完成但不代表真实完成"的情况？
```
